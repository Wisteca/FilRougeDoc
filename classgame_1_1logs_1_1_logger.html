Detailed Description

Singleton Logger.

Because this class inherit from std::ostream the way to log something is like that :

Logger::log() << info << "System started." << endlog;

The aim is to bring a maximum of flexibility when logging something and to log like in cout for example.

A log may always begin by an identificator

  • info
  • warning
  • severe

If it isn't, the identificator of the last log is used.

To terminate a log, use the endlog identificator. The log isn't saved before this call, so for example :

Logger::log() << info << "Hello all,"; Logger::log() << " I'm a log"; Logger::log() << endlog; will create only one log.

It's not possible to log something with more than one severity, so :

Logger::log() << info << "This is info"; Logger::log() << warning << "This is warning"; // The info log is logged (endlog is automatically called) Logger::log() << endlog; // Warning is logged

This class is based on : https://stackoverflow.com/questions/772355/how-to-inherit-from-stdostream.

Detailed Description

Console manager class. An instance of this can manage all display using its own render thread.

Do not write to console while an object of this class is running. Multiple threads that access to std::cout can do weird things.

This class has a pixels buffer where pixels are char. It cannot be accessed directly. To display something create a Sprite or MultiSprite object and add it to the ConsoleEngine object.

The screen is updated regarding on the FPS value used to build the object. To avoid re-drawing some unchanged stuff, all Sprite have a flag that indicate if they need to be drawn. If no sprite has to be drawn, the screen doesn't update.

Usage example :

ConsoleEngine ce(Vec2<int>(140, 40), 1.); // Specify the size used to render and eventually the framerate
ce.startRendering(); // Start to render to the screen, for now, there is nothing to display
Sprite* monster = new Sprite("assets/monster.sprite"); // Load a monster sprite from a file
ce.addSprite(monster); // Add the sprite to the renderer.
// Note : if a sprite must be drawn in front of another, you can specify a layer.
// The monster will now be displayed to the screen.
// You can move it like this :
while(1)
{
this_thread::sleep_for(chrono::milliseconds(1000));
monster->setPosition(monster->getPosition() + Vec2<int>(1, 2));
// In this example the monster will move one to the right and two to the bottom each second.
// Because the console engine used its own thread, no need to call a method render() or something like that.
}

To remove a Sprite that do not need to be drawn anymore, use removeSprite()

The ConsoleEngine can be used to read inputs too. It uses OS specific code to capture inputs directly. When a new char is captured, an event ConsoleKeyPressedEvent is pushed to the EventManager class, it can next be processed by the main thread on call of processEvents(). The ConsoleKeyPressedEvent has a method getChar() to get the char corresponding to the key.

Detailed Description

Abstract Controller class that can manage a GameObject.

The Controller class manages a GameObject. It's the Controller that gives "life" to an object by giving it a behavior.

The Controller is binded to a GameObject and will usually manage its creation and destruction.

The ticks() method is called by the GameManager singleton each 100ms to give possibility to the Controller executing its behavior.

If a Controller has finished operations (example : its binded entity died, the object has been used), it can return true when the GameManager call the hasFinished() method. The GameManager will remove it from the Controller list and the destructot will be called.

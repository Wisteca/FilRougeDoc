Detailed Description

Provide utility methods to compute an attack.

This class is designed to be the more flexible possible, it can be see like a C structure with some processing methods on it. The class provides a generic attack pipeline that has no obligation to be exaclty followed and this is why some members are public.

Usually the Item, Entity or something/someone else who wants to do damages will create an AttackComputer object, configure it (by using provided methods or simply setting public members) and send it to the Entity::endureAttack(AttackComputer) method of target.

A copy of this object will keep the same references to attacker and target (copy constructor isn't overrided). So if an item has to attack more than one ennemy, it is possible to create multiple copy objects and only call setTarget() method to change the target.

void game::core::AttackComputer::prepareAttack ( )

Compute the power of the attack.

It is needed to call configureAttack() and setAttacker() before.

This method will add a bonus on damages regarding of the attacker attack skill. Then it will make a random to determine if the attack is critical, if it is, damages are multiplied by a critical factor.

◆ targetDodge()

void game::core::AttackComputer::targetDodge ( )

Determine if the target can dodge the attack.

It is needed to call setAttacker() and setTarget() before.

This method will make a random between 0 and the dodge value of target, then it will compare this value to the attack skill of the attacker, if the random value is greater, the dodge is a total success and no damage goes to the target. Otherwise, if the value is near the attack skill, the dodge is a little success and the damages will be decreased regarding of the difference. If the value is far, the dodge failed and the damages stays the same.
